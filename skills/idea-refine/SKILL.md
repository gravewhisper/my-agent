# Idea Refine

## Overview
Turn a vague idea into a sharper proposal worth planning.

## When to Use
- Early exploration
- Ambiguous feature ideas
- Too many directions and no selection criteria

## Workflow
1. Restate the idea in one sentence.
2. Identify the user, problem, and desired outcome.
3. Generate 3-5 materially different directions.
4. Compare them on value, feasibility, risk, and differentiation.
5. Surface assumptions that need validation.
6. Recommend one direction and explain why.
7. If the direction is chosen, hand off to `spec-driven-development`.

## Common Rationalizations
| Rationalization | Reality |
|---|---|
| We can figure it out while coding | Unclear goals create thrash and rework. |
| There is only one obvious solution | The first idea is usually not the only good one. |

## Red Flags
- Solution chosen before the problem is clear
- No explicit assumptions
- Multiple unrelated goals bundled together

## Verification
- [ ] Problem, user, and outcome are explicit
- [ ] Alternatives were considered
- [ ] Next step is clear: stop, validate, or spec the idea
